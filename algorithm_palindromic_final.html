From the previous blog, we find out that there must be better solutions.

Let's go to the topic directly.

Algorithm idea: Observe features of palindromic string, it is clear that there is a center. The center may be one character or two. So we can use this feature as entry point.

Code talks:

/**
     * Third method. Time complexity: O(n^2), Space complexity: O(1). Algorithm
     * idea: given a center, either one letter or two letter, and find longest
     * palindrom.
     * @param string
     * @return longestPalindromicSubstring
     * */
    private String getLongestPalindromicSubstring_3(String string) {
        if (string.isEmpty()) {
            return null;
        }

        if(string.length() == 1) {
            return string;
        }

        String longestPalindromicSubstring = string.substring(0, 1);
        for (int i = 0; i < string.length()-1; i++) {
            String temp = getLongestPalindromWithACenter(string, i, i).length() 
                    >getLongestPalindromWithACenter(string, i, i+1).length() 
                    ? getLongestPalindromWithACenter(string, i, i)
                    :getLongestPalindromWithACenter(string, i, i+1);
            if(temp.length() > longestPalindromicSubstring.length()) {
                longestPalindromicSubstring = temp;
            }
        }

        return longestPalindromicSubstring;
    }

    /**
     * given a center, either one letter or two letter, and find longest
     * palindrom.
     * @param string
     * @param begin
     * @param end
     * @result longestPalindromWithACenter
     * */
    private static String getLongestPalindromWithACenter(String string,
            int begin, int end) {
        while ((begin >= 0) && (end <= string.length() - 1)
                && ((string.charAt(begin)) == (string.charAt(end)))) {
            begin--;
            end++;
        }
        return string.substring(begin+1, end);
    }
