Word Break, to be continued....

If we just think about implementation with java code, and using pattern matching may be the simplest method. In this way, we can solve this problem by building a DFA in O(2^m) and executing it in O(n). See here;

Talk is cheap, show the code:


public static void main(String[] args) {
        HashSet<String> dict = new HashSet<String>();// set has no duplicate
                                                        // elements.
        dict.add("go");
        dict.add("goal");
        dict.add("goals");
        dict.add("special");

        StringBuilder sb = new StringBuilder();

        for (String s : dict) {
            sb.append(s + "|");// StringBuider is better in this case.(x|y means
                                // matching x or y.)
        }

        String pattern = sb.toString().substring(0, sb.length() - 1);
        pattern = "(" + pattern + ")*";// (pattern)* means matching this pattern
                                        // zero or more times
        Pattern p = Pattern.compile(pattern);// Pattern class has no public
                                                // constructor and we must use
                                                // its public static compilation
                                                // method.
        Matcher m = p.matcher("goalspecial");// Just like pattern, and we can
                                                // get matcher instance by
                                                // invoking Pattern object's
                                                // matcher() method.

        if (m.matches()) {// try to match the whole area.
            System.out.println("match");
        }
    }

To sum up, class Pattern and Matcher seem very useful to deal with strings. Extra time should be spent on the source code and also need to try applying it on usual practice.
